# AWS Basic

## 📌 EC2, RDS, S3, Route53, ELB, CloudFront, Lambda

### 🔹 AWS SAA

- 서비스의 역할, 의미, 기능에 대해 기억하기

### 🔹 EC2(Elastic Compute Cloud)란?

- EC2 = 컴퓨터를 빌려서 원격으로 접속해 사용하는 서비스

### 🔹 RDS(Relational Database Service)란?

- RDS = AWS의 RDBMS 서비스

### 🔹 S3(Simple Storage Service)란?

- S3 = AWS의 파일 저장 서비스
  - ex. 구글 드라이브
- S3의 부가적인 기능 : 정적 웹사이트 호스팅
  - 웹 서비스를 다른 사용자들도 쓸 수 있게 인터넷에 배포하는 것

### 🔹 Route 53이란?

- Route 53 = AWS의 도메인 발급 및 관리 서비스(DNS)
- 도메인 = 인터넷 주소
- DNS(Domain Name System) : IP 주소와 문자열 사이의 변환 작업을 해주는 시스템

### 🔹 ELB(Elastic Load Balancer)란?

- ELB = AWS의 로드밸런서
  - 로드밸런서 : 트래픽을 적절하게 분배해주는 장치
- 서버를 2대 이상 가용할 때 ELB를 필수적으로 도입함

### 🔹 CloudFront란?

- CloudFront = AWS의 컨텐츠(파일, 동영상 등) 전송 서비스
- CDN : 컨텐츠의 복사본을 전세계 곳곳에 저장해두는 저장소를 운영함
  - 이를 통해 사용자는 데이터를 빠르게 받을 수 있음

### 🔹 Lambda란?

- Lambda = AWS의 서버리스 서비스
  - 서버는 AWS가 관리하고, 개발자는 코드만 작성하면 실행되는 서비스

## 📌 리전, 가용성, 가용 영역, 다중 AZ, 단일 AZ

### 🔹 리전(Region)이란?

- 리전 = 지리적으로 나눠서 설치해둔 각 데이터 센터
  - 각 데이터 센터에는 AWS 서비스들이 있음(EC2 등)

### 🔹 가용성(Availability)

- 가용성 = 시스템이 서비스를 정상적으로 제공할 수 있는 가능성
  ```markdown
  가용성 = up time / (up time + down time)
  ```
- 고가용성 시스템 : 서비스 장애가 발생할 가능성이 낮은 시스템
- 가용성은 서비스의 정상 가동률(%)로 표현됨
  - ex. 가용성 99.99% = 99.99% 시간을 정상적으로 사용 가능한 시스템
  - 이는 1년에 53분정도 서비스가 다운될 수 있음

### 🔹 가용 영역(Availability Zone)이란 ?

- 가용 영역(AZ) = 하나의 리전 안에서 물리적으로 분리된 데이터 센터
- 자연 재해로 인해 데이터 센터가 물리적으로 파괴될 가능성을 고려해서, 물리적으로 분리된 데이터 센터를 만든 것

### 🔹 다중 AZ(Multi-AZ)란?

- 다중 AZ = 하나의 리전 안에서 서로 다른 AZ에 시스템을 분산 배치하는 구성 방식
- 같은 서비스 구성 요소를 여러 AZ에 동시에 배치해서 특정 AZ에 장애가 발생해도 서비스가 중단되지 않도록 만들기 위함
- 고가용성을 위해선 다중 AZ가 필요

### 🔹 단일 AZ(Single-AZ)란?

- 단일 AZ = 하나의 가용 영역에만 시스템을 배치하는 굿어 ㅂ아식
- 비용이 저렴한 장점
