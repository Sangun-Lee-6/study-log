# 1_Docker 기본 개념

## 1. 도커를 많이 사용하는 이유

### 🔹 이식성

- 도커의 가장 큰 장점은 이식성
- 이식성 : 특정 프로그램을 다른 곳으로 쉽게 옮겨서 설치하고 실행할 수 있는 특성

### 🔹 예시

- 내 컴퓨터에 잘 설치된 MySQL이 다른 컴퓨터에서는 설치가 안될 수 있음
    - 버전이 다르거나, OS가 다르거나, 보안 프로그램에서 충돌이 발생하는 등 다양한 이유로 같은 프로그램이 다른 컴퓨터에서 정상적으로 설치되지 않음
- 이러한 문제를 해결하는 도구가 도커
    - 도커를 사용하면 명령어로 어떤 컴퓨터에서든 프로그램을 설치하고 실행 가능

### 🔹 도커를 사용했을 때 장점

- 설치 과정의 간편함(명령어 기반)
- 항상 같은 프로그램을 설치 가능(버전, 환경 설정 등)
- 프로그램 간 충돌 X
    - ∵ 각 프로그램이 독립적인 환경에서 실행되므로

## 2. IP와 Port의 개념

### 🔹 IP

- 네트워크에서 특정 컴퓨터를 가리키는 주소
- ex. IP 주소 : `13.250.15.132`

### 🔹 Port

- 한 컴퓨터 안에서 실행되고 있는 특정 프로그램의 주소
    - 한 컴퓨터 안에선 여러 개의 프로그램이 실행되고 있음
    - 외부에선 IP 주소만으로는 특정 프로그램과 통신할 수 없음 → 포트 번호까지 알아야 통신 가능
- ex. `13.250.15.132:3000`
    - 여기서 `:3000`이 포트 번호

### 🔹 브라우저에서 포트 번호를 입력하지 않는 이유

- 도메인 주소는 IP 주소만 나타내지만, 통신이 가능하려면 포트 번호까지 알아야함
- 브라우저는 기본적으로 80번 포트로 통신을 보내도록 설정이 되어 있어서 생략되어 있는 것

### 🔹 well-known port

- 포트 번호는 0번~65,535번까지 사용 가능
- 그 중 0번~1,023번까지 포트는 주요 통신을 위한 규약에 따라 이미 정해짐
- 이렇게 규약을 통해 역할이 정해진 포트 번호를 잘 알려진 포트(well-known port)라고 부름
- well-known port 예시
    - 22번 : SSH(Secure Shell Protocol), 원격 접속을 위한 포트 번호
    - 80번(HTTP) : HTTP로 통신을 할 때 사용
    - 443번(HTTPS) : HTTPS로 통신을 할 때 사용

## 3. 도커, 컨테이너, 이미지

### 🔹 도커란?

- 컨테이너를 사용해서 각 프로그램을 분리된 환경에서 실행, 관리할 수 있게 해주는 도구
- 우선 이해할 개념은 2가지
    1. 컨테이너
    2. 분리된 환경

### 🔹 컨테이너란?

- 하나의 컴퓨터 안에서 독립적인 컴퓨터 환경을 구성한 것, 일종의 미니 컴퓨터
- 각 컨테이너마다 프로그램을 별도로 설치할 수 있고, 이는 서로 컨테이너끼리 독립적
- 호스트 컴퓨터 : 컨테이너를 포함하고 있는 컴퓨터
    - 즉, 호스트 컴퓨터 안에 여러 컨테이너가 구성될 수 있음

### 🔹 컨테이너의 독립성

- 컨테이너는 독립적인 컴퓨터 환경
- 여기서 독립적으로 관리되는 것
    1. 디스크 : 각 컨테이너마다 서로 저장 공간을 갖고 있음, 일반적으로 다른 컨테이너의 내부 파일에 접근 불가
    2. 네트워크(IP, Port) : 각 컨테이너마다 고유한 네트워크를 갖고 있음, 컨테이너마다 IP 주소 있음
- CPU, 메모리는 호스트 컴퓨터의 자원을 나눠서 사용함

### 🔹 이미지란?

- 이미지 ≒ 닌텐도 게임기의 칩
    - 여러 칩을 게임기에 꽂아서 다양한 게임을 할 수 있음
- Express.js 서버를 이미지로 만들어서 도커로 실행시키면, Express.js 서버가 컨테이너 환경에서 쉽게 실행됨
- 이미지에는 프로그램을 실행할 때 필요한 모든 것이 포함됨
    - 프로그램 코드 + 프로그램을 실행할 때 필요한 설치 과정, 설정, 버전 정보 등
