- 정렬, 투포인터
- 하나는 고정하고, 나머지 2개는 Two Sum 문제로 해결
- 정렬 : 중복 제거 및 투포인터 이동을 위해 사용
- 투 포인터 : 나머지 두 숫자를 `left`, `right`로 탐색
- 시간 복잡도 : O(n^2)
  - 정렬 : O(n log n)
  - 탐색 : O(n^2)
  - nums.length ≤ 3000이라서 O(n^3)은 시간 초과
- HashMap이 아니라 투 포인터를 사용한 이유
  - 3Sum에서는 중복 triplet 제거가 핵심
  - 중복 제거가 깔끔하므로, 투포인터 사용

```python
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        nums.sort()
        result = []

        for i, n in enumerate(nums):
            if i > 0 and nums[i] == nums[i - 1]: # ⚠️ 개선 포인트 1
                continue

            if n > 0: # ⚠️ 개선 포인트 2
                break

            target = n * (-1)
            left = i + 1
            right = len(nums) - 1

            while left < right:
                sum = nums[left] + nums[right]

                if sum == target:
                    triplets = [n, nums[left], nums[right]]
                    result.append(triplets)

                    left += 1
                    right -= 1 # ⚠️ 개선 포인트 3

										# ⚠️ 개선 포인트 4
                    while left < right and nums[left] == nums[left - 1]:
                        left += 1

                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1

                elif sum < target:
                    left += 1
                else:
                    right -= 1

        return result
```

- ⚠️ 개선 포인트 1
  - 같은 n은 한 번만 기준값으로 사용
  - 같은 n은 같은 triplet을 만들 수 있으므로 건너뜀
- ⚠️ 개선 포인트 2
  - 정렬되어 있는데 n > 0 이라면 뒤의 값들도 모두 양수이고, 세 수를 더해서 양수가 나올 수 없으므로 생략
- ⚠️ 개선 포인트 3
  - 조합을 찾은 뒤 양쪽 포인터 둘 다 이동해서 더 빨리 탐색
- ⚠️ 개선 포인트 4
  - 같은 `left`, `right` 중복 제거
